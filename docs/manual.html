🎬 Video AI Toolkit

Manual completo — dê ao seu agente de IA o ciclo inteiro sobre vídeos: baixar → assistir → transcrever → editar. Tudo local, tudo grátis, pt-BR primeiro.

Visão geral

O toolkit reúne 2 skills, 2 agentes e 2 aplicativos que funcionam sozinhos ou encadeados por linguagem natural em qualquer agente que leia SKILL.md (Claude Code, Codex, Gemini CLI, Cursor…):

Instalação

Pré-requisito: Python 3.10+ (Microsoft Store no Windows; brew/apt no resto).

# Windows (PowerShell)
git clone https://github.com/JohnsonChenx/video-ai-toolkit
cd video-ai-toolkit
powershell -ExecutionPolicy Bypass -File install.ps1           # stack de vídeo
powershell -ExecutionPolicy Bypass -File install.ps1 -Escriba  # + GUI do Escriba
powershell -ExecutionPolicy Bypass -File install.ps1 -Force    # sobrescreve skills existentes

# macOS / Linux
bash install.sh          # (--force para sobrescrever)

O instalador configura, na ordem:

  1. ffmpeg — via winget/brew/apt, sempre das fontes oficiais (não redistribuímos GPL)
  2. yt-dlp + claude-real-video + faster-whisper — via pip
  3. Deno — runtime JS que o YouTube exige desde 2026; no Windows o winget o instala fora do PATH, então o caminho é registrado em %APPDATA%\yt-dlp\config
  4. PYTHONUTF8=1 permanente (Windows) — sem isso, consoles cp1252 quebram ferramentas no meio da execução
  5. Skills e agentes copiados para ~/.claude/
⚠ Antivírus que intercepta HTTPS (Avast e afins) Se o pip falhar com erro de SSL: python -m pip install --user pip-system-certs e rode o instalador de novo. O instalador já usa --source winget para evitar a fonte msstore, que quebra pelo mesmo motivo.

Skill youtube — baixarSKILL

Baixa vídeo sempre em MP4 (prefere H.264+AAC; outros codecs são remuxados) e áudio sempre em MP3 na melhor qualidade. Se autoinstala: verifica yt-dlp/ffmpeg/Deno e corrige o que faltar.

Comandos que ela usa por baixo

# Vídeo — sempre MP4
yt-dlp -f "bv*[vcodec^=avc1]+ba[acodec^=mp4a]/bv*+ba/b" --merge-output-format mp4 --remux-video mp4 --no-playlist -o "PASTA\%(title)s.%(ext)s" "URL"

# Áudio — sempre MP3, melhor qualidade
yt-dlp -x --audio-format mp3 --audio-quality 0 --no-playlist -o "PASTA\%(title)s.%(ext)s" "URL"

# Playlist inteira (subpasta numerada)
-o "PASTA\%(playlist_title)s\%(playlist_index)s - %(title)s.%(ext)s"   # e sem --no-playlist
Atenção com URLs de playlist watch?v=...&list=... baixa a playlist INTEIRA sem --no-playlist. A skill só omite a flag quando o usuário pede a playlist explicitamente.

Falhou com "signature", "extractor" ou HTTP 403? Quase sempre é yt-dlp desatualizado: pip install -U yt-dlp.

Skill claude-real-video — assistirSKILL

O agente não tem input de vídeo — esta skill dá um: extrai keyframes por detecção de cena (com dedup em 2 canais), transcreve (legendas existentes → faster-whisper) e gera contact sheets 3×3 que economizam ~9× tokens de imagem.

crv "<url-ou-arquivo>" -o crv-out --grid --why "o que o usuário quer saber"
# vídeos longos:  --max-frames 60
# sem fala:       --no-transcribe
# entrevistas:    --speakers   (pip install "claude-real-video[speakers]")

O agente lê primeiro MANIFEST.txt (resumo + transcrição integral), depois os grids em grids/, e responde citando timestamps de transcript.json.

watch vs. claude-real-video — quando usar cada um

SituaçãoMelhor escolha
Olhada rápida em vídeo com legendas nativas/watch — transcript em segundos, sem GPU
"O que acontece aos 12:40?" (trecho focado)/watch --start/--end — frames densos na janela
Análise profunda, resumo com evidência visualcrv --grid — dedup superior + grids econômicos
Entrevista/podcast (quem fala o quê)crv --speakers — diarização local
Conteúdo sensívelcrv — 100% local, nada sai da máquina

Agente escriba — transcreverAGENTE

Transcrição de áudio/vídeo em pt-BR com separação de falantes (WhisperX + pyannote), 100% local. Instala a própria pipeline do zero ("prepara o ambiente de transcrição") e guia o usuário nos únicos passos manuais: conta HuggingFace + aceite dos 3 modelos de diarização + token.

Fluxo de decisão automático

  1. Fonte é YouTube? Tenta legendas nativas primeiro (legendas.py, pt-BR preferido): palestra de 2 h = 2 segundos, contra ~25 min de GPU. WhisperX só quando precisa de falantes separados, não há legenda, ou a legenda automática é ruim.
  2. Estimativa de tempo (obrigatória): mede a duração via ffprobe e pergunta — "Rápido (medium, ~10% da duração) ou Máxima precisão (large-v3, ~20%)?" Nunca escolhe sozinho; reporta tempo real vs. estimado ao final.
  3. Qualidade de áudio (automática): audio_quality.py mede o SNR (fala = p95 do RMS em janelas de 0,5 s; ruído = p10, revelado nas pausas) e detecta voz abafada por bandas. Decide sozinho:
    DiagnósticoTratamento
    SNR ≥ 25 dBnenhum (denoise em áudio limpo borra consoantes)
    12–25 dBRNNoise via ffmpeg (zero instalação; modelo baixa sozinho)
    < 12 dBDeepFilterNet se instalado, senão RNNoise
    Voz abafadaResemble Enhance se instalado (dereverb)
    Gera <nome>.limpo.wav sem tocar no original; falhas caem em fallback silencioso. Controle: --denoise auto|off|forcar.
  4. Transcrição: WhisperX com diarização ([SPEAKER_00]…), saídas .txt/.srt/.json (timestamps por palavra).
  5. Formato (pergunta obrigatória): "Resumo Estruturado ou Ata de Reunião?" — nunca adivinha.
    • Resumo Estruturado: Essência (1 frase) · Tema · Pontos-chave com evidência · Implicações · Ações · Citações → .topicos.md
    • Ata: Data/horário · Participantes (SPEAKER→nome) · Pauta · Discussão · Decisões · tabela Prazo|Responsável|Ação · Questões em aberto → .ata.md
    Regra comum: dado ausente vira "não mencionado"/"a definir" — nada é inventado.

Flags e comandos principais

python transcrever.py <arquivo_ou_pasta>
  [--modelo medium|large-v3|...]    # default: medium
  [--falantes N]                    # N exato melhora muito a diarização
  [--recursivo] [--forcar]          # modo pasta/lote
  [--denoise auto|off|forcar]       # análise de qualidade automática
  [--device cuda|cpu] [--batch-size N] [--compute-type ...]

python legendas.py "URL" --saida PASTA     # legendas nativas (pt→en) em .txt/.json
python audio_quality.py analisar ARQ --json # só medir; "tratar" aplica o denoise

Agente editor — editarAGENTE

Edição de vídeo por conversa: transforma bruto de gravação (erros, silêncios, refações) em vídeo limpo, com render frame-accurate via ffmpeg (NVENC quando há GPU NVIDIA).

Duas fases + método das 3 passadas

  • FASE A — plano: transcreve (pipeline do escriba, --falantes 1), roda as 3 passadas e entrega plano-cortes.md para aprovação. Não corta nada.
  • FASE B — aplicação: só com aprovação explícita ou modo autônomo declarado ("vou dormir, termina"). Extrai os segmentos mantidos com re-encode, concatena, roda QA e entrega.
PassadaO que detectaFonte da verdade
1 — ErrosPalavra-chave falada ("corta"), refações, gaguejos, frases abandonadasTranscrição (timestamps por palavra)
2 — Silêncio realPausas acima do limiar (default 0,6 s, respiro de 0,3 s)Áudio via silencedetect — nunca a transcrição
3 — RedundânciaMesma ideia dita 2+ vezes (fica a melhor versão)Transcrição do vídeo inteiro

Extras validados em produção: cortes verticais 9:16 com crop calibrado por QA visual de frames, e legenda karaokê (ASS \kf por palavra a partir do JSON do WhisperX — cuidado: a linha Dialogue: tem exatamente 9 campos).

Extensões automáticas E1–E3

Detectam o gatilho, agem e informam — sem perguntar. O pré-check classifica a máquina (FRACA = sem GPU ou RAM < 16 GB) e é isso que dispara os fallbacks.

Ext.PrimárioFallback automáticoGatilho do fallback
E1 Motion graphicsRemotion (React → MP4/WebM alpha)Motion Canvas (MIT)licença Remotion (>3 pessoas), pedido "só open source", setup falhou
E2 Verificação nota 0-100crv + o próprio agente avalia (rubrica: emendas 30 + áudio 25 + cortes 25 + layout 20; entrega só com ≥ 85)Gemini API free tiermáquina FRACA, vídeo > 15 min, crv falhou
E3 Imagens p/ motionsGemini API free tier (prompt contextual ao SRT)ComfyUI local (SDXL, roda em 6 GB VRAM)cota esgotada (429) ou sem chave
Privacidade vence gatilho Conteúdo sensível/de cliente nunca vai para API externa, mesmo com gatilho ativo. E sem GEMINI_API_KEY nada bloqueia — o fluxo segue local. Para ativar os fallbacks, veja Configurar GEMINI_API_KEY (2 minutos, grátis).

App Baixador (interface web)APP

apps/baixador/ABRIR.bat abre http://localhost:8765: cole o link, escolha Vídeo (MP4) ou Áudio (MP3), selecione a pasta com o seletor nativo do Windows, acompanhe a barra de progresso. Playlist inteira por checkbox; log técnico expansível; botão de cancelar. Mantenha a janela preta (servidor) aberta durante o uso — os botões param de responder sem ela. Alternativas CLI: BAIXAR.bat (menu) e baixar.ps1.

App Escriba (GUI de transcrição)APP

apps/escriba/Iniciar Escriba.bat (Windows; deps via install.ps1 -Escriba):

  • Arrasta e solta arquivo ou pasta inteira → transcrição com [SPEAKER_XX] (a análise de qualidade + denoise rodam automaticamente por baixo)
  • Ditado global: segure Ctrl+Alt+D em qualquer programa, fale, solte — o texto aparece no cursor
  • Botões "Resumo Estruturado" e "Ata de reunião" — via Ollama local (grátis) ou Claude API (ANTHROPIC_API_KEY)

Frases prontas (a cola)

# Pipeline completo numa frase (com legenda nativa, sai em segundos)
"Baixa essa aula do YouTube, transcreve e me entrega um resumo estruturado: URL"

# O agente assiste (slides/demos importam)
"Assiste essa aula e resume incluindo o que aparece nos slides: URL"

# Zoom em trecho
"O que acontece dos 15 aos 20 minutos desse vídeo? URL"

# Curso inteiro
"Baixa o áudio da playlist inteira e transcreve a pasta toda"

# Reunião com ata
"Transcreve identificando quem fala"  →  escolher "Ata"

# Velocidade vs precisão (evita a pergunta)
"...transcreve rápido"            → medium (~10% da duração)
"...com máxima precisão"          → large-v3 (~20%)

# Tratamento de áudio
"...sem tratamento de áudio"      → --denoise off
"...força a limpeza"              → --denoise forcar

# Edição
"Edita esse bruto, corta os erros e silêncios"        → FASE A (plano)
"Plano aprovado, aplica"                              → FASE B
"Faz cortes de TikTok do vídeo final"                 → 9:16 + karaokê

Configurar a GEMINI_API_KEY (opcional, grátis)

A chave ativa os fallbacks automáticos do agente editor: verificação de vídeo em máquinas fracas ou vídeos longos (E2) e geração de imagens para motions (E3). Sem ela, tudo continua funcionando no modo local — a chave só destrava os upgrades. Leva ~2 minutos:

  1. Acesse o Google AI Studio: aistudio.google.com/apikey e entre com qualquer conta Google (não precisa de cartão de crédito).
  2. Clique em "Create API key" (ou "Criar chave de API"). Se pedir um projeto do Google Cloud, aceite a opção de criar um novo automaticamente.
  3. Copie a chave — começa com AIza.... Trate como senha: ela sai da tela e não deve ir para arquivos de código, chats ou repositórios.
  4. Grave como variável de ambiente: 
    # Windows (PowerShell) — permanente para novas janelas:
setx GEMINI_API_KEY "AIza...sua-chave..."
# e também na sessão atual (setx não afeta janelas já abertas):
$env:GEMINI_API_KEY = "AIza...sua-chave..."

# macOS / Linux — adicione ao ~/.zshrc ou ~/.bashrc:
export GEMINI_API_KEY="AIza...sua-chave..."
  5. Verifique (deve listar modelos disponíveis): 
    # Windows (PowerShell)
$k = [Environment]::GetEnvironmentVariable("GEMINI_API_KEY", "User")
(Invoke-RestMethod "https://generativelanguage.googleapis.com/v1beta/models?key=$k").models | Select-Object -First 3 name

# macOS / Linux
curl -s "https://generativelanguage.googleapis.com/v1beta/models?key=$GEMINI_API_KEY" | head -20
Limites do tier gratuito O free tier tem cotas por dia e por minuto que variam por modelo (generosas para uso pessoal). Quando a cota esgota, a API responde HTTP 429 — e é exatamente esse sinal que o editor usa para transbordar automaticamente para o ComfyUI local (E3) ou seguir no método local (E2). Ou seja: estourar a cota nunca quebra o fluxo.
Segurança da chave Nunca commite a chave em repositório nem cole em chats. Se vazar, revogue e gere outra na mesma página do AI Studio. O toolkit só a lê da variável de ambiente — nenhum arquivo do projeto a armazena.

Privacidade e licenças

Privacidade: todo o processamento (download, frames, transcrição, diarização, denoise, edição) roda na sua máquina.
